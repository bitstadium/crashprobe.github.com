Introduction

Every day more and more apps and new versions of existing apps are being released. While these apps are getting more sophisticated, it is also important to make sure problems and especially crashes can be fixed very quickly.

A lot of services and frameworks have emerged in the past years to help developer solve these crashes more quickly. HockeyApp launched in May 2011 and is using and supporting the development of the proven and reliable Open Source framework PLCrashReporter ever since.

In our effort to ensure best possible and most accurate crash reports, we decided to create and open source CrashProbe. The CrashProbe open source project that provides a set of currently 21 test crashes that cover a wide area of different crashing scenarios on iOS and OS X.

Two components are required to provide accurate crash reports:

  1. An async-safe crash reporting framework that can be embedded into apps to catch crash details
  2. Symbolication process that adds more details to a crash report including class names, filenames and line numbers
