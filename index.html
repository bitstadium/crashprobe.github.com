Introduction

Every day more and more apps and new versions of existing apps are being released. While these apps are getting more sophisticated, it is also important to make sure problems and especially crashes can be fixed very quickly.

CrashProbe is an open-source crash reporting benchmark that tests the accuracy and reliability of popular iOS and Mac OS X crash reporting services. Each service is tested against realistic instrumented crashes, with the resulting crash reports validated for accuracy and usefulness in diagnosing crashes.

CrashProbe is sponsored by HockeyApp. Launched in May 2011, HockeyApp has long used and supported the ongoing development of PLCrashReporter, a proven and reliable Open Source crash reporting framework.

In our effort to ensure best possible and most accurate crash reports, we decided to create and open source a test suite that we could use to test PLCrashReporter, and improve HockeyApp. The result, CrashProbe, provides a set of instrumented test cases that cover a wide area of different crashing scenarios on iOS and OS X.

Two components, working in concert, are required to provide accurate crash reports:

1. An async-safe crash reporting framework that can be embedded into apps to accurately capture crash details.

2. A server-side symbolication process that adds more details to a crash report including class names, filenames and line numbers.

Open Source

You can perform all tests yourself with any crash reporting framework on iOS and OS X and compare the quality of the symbolicated crash reports yourself. The source code is available under the MIT license on GitHub: https://github.com/bitstadium/CrashProbe
